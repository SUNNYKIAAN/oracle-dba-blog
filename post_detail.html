How to Configure Oracle Data Guard

Posted on July 25, 2025

Oracle Data Guard ensures high availability, data protection, and disaster recovery for enterprise data. In this guide, we will walk through setting up Oracle Data Guard from scratch.

Step 1: Enable Force Logging

SQL> ALTER DATABASE FORCE LOGGING;

Step 2: Create Standby Redo Logs

SQL> ALTER DATABASE ADD STANDBY LOGFILE GROUP 4 ('/u01/app/oracle/oradata/redo04.log') SIZE 500M;

Step 3: Configure Listener and TNS

Update listener.ora and tnsnames.ora on both primary and standby.

Step 4: Enable Redo Transport

SQL> ALTER SYSTEM SET LOG_ARCHIVE_DEST_2='SERVICE=standby LGWR ASYNC VALID_FOR=(ONLINE_LOGFILES,PRIMARY_ROLE) DB_UNIQUE_NAME=standby';

Repeat similar steps on the standby database and test the configuration thoroughly.
